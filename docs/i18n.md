# 国際化（i18n）ガイド

riverapp.jp の多言語対応の仕組みと翻訳の追加方法のドキュメントです。

---

## 概要

- **対応言語**: 日本語（デフォルト）、英語
- **実装方式**: React Context ベースのカスタム実装（外部ライブラリ不使用）
- **言語切り替え**: Navbar の言語切り替えボタンで ja/en をトグル
- **永続化**: Cookie（`locale`、365日間保持）
- **フォールバック**: 翻訳キーが見つからない場合は日本語を表示

---

## ファイル構成

```
src/i18n/
├── config.ts              # ロケール設定・翻訳キーマッピング
├── context.tsx            # I18nProvider・翻訳フック
└── translations/
    ├── ja.json            # 日本語翻訳（静的import、バンドルに含まれる）
    └── en.json            # 英語翻訳（動的import、必要時にロード）
```

## 翻訳の2つのレイヤー

riverapp.jp の翻訳は **2つのレイヤー** で構成されています。

### レイヤー 1: サイト共通テキスト（translations/*.json）

Navbar、フィルター、Hero セクションなど **サイト全体で共通の UI テキスト** を管理します。

```json
// translations/ja.json
{
  "hero": {
    "label": "個人開発者",
    "heading1": "ゲームも、便利ツールも、"
  },
  "filter": {
    "all": "すべて",
    "game": "ゲーム",
    "app": "アプリ"
  },
  "tags": {
    "multiLanguage": "多言語対応",
    "iosApp": "iOSアプリ"
  }
}
```

コンポーネントでは `useTranslation()` フックで使用:

```tsx
const t = useTranslation();
<p>{t("hero.label")}</p>
<p>{t("filter.game")}</p>
```

変数補間も対応:

```tsx
t("appCard.viewDetail", { name: "HEXLIDE" })
// → "HEXLIDEの詳細を見る"
```

### レイヤー 2: アプリ個別テキスト（JSON の `en` フィールド）

各アプリの説明文、タグ、リンクラベルなど **アプリ固有のテキスト** は、アプリの JSON ファイル内の `en` フィールドで管理します。

```json
// src/data/apps/hexlide.json
{
  "name": "HEXLIDE",
  "shortDescription": "HEXLIDEは、六角形グリッドの対戦ゲームです。",
  "en": {
    "shortDescription": "HEXLIDE is a hexagonal grid board game.",
    "linkLabels": {
      "今すぐプレイ": "Play Now",
      "プライバシーポリシー": "Privacy Policy"
    },
    "documents": {
      "appDisplayName": "HEXLIDE",
      "about": { ... }
    }
  }
}
```

コンポーネントでは `useLocalizedApp()` フックで自動マージ:

```tsx
function AppHeader({ app: rawApp }) {
  const app = useLocalizedApp(rawApp);
  // locale === "en" の場合、app.shortDescription は英語版に自動切り替え
  return <p>{app.shortDescription}</p>;
}
```

## 翻訳フック一覧

| フック | 返り値 | 用途 |
|--------|--------|------|
| `useI18n()` | `{ locale, setLocale, t }` | フルアクセス |
| `useTranslation()` | `t` 関数 | サイト共通テキストの翻訳 |
| `useLocale()` | `{ locale, setLocale }` | 言語切り替え |
| `useLocalizedApp(app)` | ローカライズ済み `App` | アプリデータの翻訳適用 |

## 翻訳キーマッピング（config.ts）

日本語のタグやラベルを翻訳キーに変換するためのマッピングが定義されています。

### TAG_TRANSLATION_MAP

日本語タグ → 翻訳キーの対応。`Tag.tsx` コンポーネントで自動適用されます。

```typescript
{
  "多言語対応": "tags.multiLanguage",
  "英語対応": "tags.englishSupport",
  "iOSアプリ": "tags.iosApp",
  // ...
}
```

### STORE_BUTTON_LABEL_MAP

ストアボタンのラベル翻訳。

```typescript
{
  "Webアプリ": "Web App",
  "App Store — 準備中": "App Store — Coming Soon",
}
```

### STORE_LABEL_MAP

ストアグループのラベル → 翻訳キーの対応。

```typescript
{
  "スマホアプリをダウンロード": "storeLabels.downloadMobileApp",
  "今すぐブラウザでプレイ": "storeLabels.playInBrowser",
  // ...
}
```

## 言語切り替えの仕組み

```
ユーザーが LanguageSwitcher をクリック
  │
  ├─ setLocale("en") 呼び出し
  │    ├─ locale 状態を更新 → React リレンダリング
  │    ├─ Cookie に "locale=en" を保存（365日）
  │    └─ loadTranslations("en") で en.json を動的ロード → キャッシュ
  │
  ├─ document.documentElement.lang = "en" を更新
  │
  └─ 全コンポーネントが新しい locale で再レンダリング
       ├─ useTranslation() → 英語翻訳を返す
       └─ useLocalizedApp() → en フィールドをマージして返す
```

## 翻訳の追加・更新方法

### サイト共通テキストを追加する

1. `src/i18n/translations/ja.json` に日本語キーを追加
2. `src/i18n/translations/en.json` に英語キーを追加
3. コンポーネントで `t("新しいキー")` を使用

```json
// ja.json
{ "newSection": { "title": "新しいセクション" } }

// en.json
{ "newSection": { "title": "New Section" } }
```

### 新しいタグを追加する

1. `src/data/tags.ts` にタグを追加
2. `src/i18n/config.ts` の `TAG_TRANSLATION_MAP` に翻訳キーを追加
3. `ja.json` / `en.json` の `tags` セクションに翻訳を追加

### アプリの英語翻訳を追加する

アプリ JSON の `en` フィールドに翻訳を記述します。翻訳可能なフィールド:

| フィールド | 説明 |
|-----------|------|
| `name` | アプリ名（通常は日英共通なので省略可） |
| `shortDescription` | 短い説明文 |
| `subtitle` | サブタイトル |
| `promoText` | プロモーションテキスト |
| `description` | Markdown 形式の詳細説明 |
| `ogDescription` | OGP 用メタディスクリプション |
| `linkLabels` | 外部リンクラベルの日→英対応 |
| `documents` | ドキュメントページの英語版 |

### 新しいストアグループラベルを追加する

1. `src/i18n/config.ts` の `STORE_LABEL_MAP` にマッピングを追加
2. `en.json` の `storeLabels` セクションに翻訳を追加

---

## 注意点

- **SSR 時は常に日本語**: `<html lang="ja">` で送信され、クライアント側で言語を切り替える
- **メタデータは日本語固定**: OGP、JSON-LD は日本語で出力される（SEO 対策として）
- **日本語がベース**: すべての翻訳は日本語をベースとし、`en` は上書き用
- **フォールバック**: 英語翻訳が見つからない場合、日本語がそのまま表示される
